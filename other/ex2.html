Exercise 2 – GitHub Setup

Create your first Git repository and push it to GitHub.

  1. Create a (Private) Git Repository

The repository should include the two files generated in Exercise 1 in a subfolder called exercise_1. The repository (i.e., the folder) should follow this naming convention dsba_<your_matriculation_number>.

Additionally, the folder must contain a README.md file that briefly describes your project and includes your matriculation number. A License should likewise be added. Find a License at choosealicense.com. Moreover, configure your .gitignore to exclude dotfiles, system/metadata files, and any temporary or cache files from being pushed to GitHub later on.

  1. Push it to GitHub

Before pushing your local repository to GitHub, you need to create a free account. This link gets you to the page where you can sign up.

After logging in, you must set up remote access from your local computer. This can be done either with an SSH key or a Personal Access Token (as outlined in the lecture slides).

Once remote access is configured, create a private GitHub repository (use the same naming convention as above) and link it as the origin for your local repository.

Once this is accomplished, invite me as a collaborator on GitHub (account: maliedvp).

  1. Bonus Task for Fast Finishers

Team up with up to two other students from the class who have also completed Steps 1 & 2. Create a shared repository (you can use one of your previously created repos and invite your teammates as collaborators). Each team member should then:

  • Clone the repository to their local machine,
  • Create a new branch, implement changes, commit those changes, and push to the remote,
  • Open a pull request from their branch to the main/master branch on GitHub.

As a group, intentionally create a merge conflict and collaboratively resolve it through the pull request process. Document the steps you took.
